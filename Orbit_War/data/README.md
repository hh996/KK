# Orbit Wars

在连续二维空间中征服围绕太阳旋转的行星。这是一款支持 2 人或 4 人对战的实时策略游戏。

## 概述

玩家从一颗母星开始，通过派遣舰队占领中立与敌方行星来争夺地图控制权。棋盘是一个 100x100 的连续空间，中心有一颗太阳。行星会绕日运行，彗星沿椭圆轨迹飞行，舰队按直线移动。游戏共进行 500 回合。结束时总舰船数（行星驻军 + 舰队舰船）最多的玩家获胜。

## 棋盘布局

- **棋盘**：100x100 连续空间，原点在左上角。
- **太阳**：中心位于 (50, 50)，半径为 10。穿过太阳的舰队会被摧毁。
- **对称性**：所有行星和彗星都以中心为轴进行四重镜像对称放置：(x, y)、(100-x, y)、(x, 100-y)、(100-x, 100-y)。这可确保不同出生点的公平性。

## 行星

每颗行星表示为 `[id, owner, x, y, radius, ships, production]`。

- **owner**：玩家 ID（0-3），或 -1 表示中立。
- **radius**：由产能决定：`1 + ln(production)`。产能越高，行星物理半径越大。
- **production**：1 到 5 的整数。每回合，已占领行星会生成对应数量的舰船。
- **ships**：当前驻军数。初始值在 5 到 99 之间（偏向较小值）。

### 行星类型

- **轨道行星**：满足 `orbital_radius + planet_radius < 50` 的行星会以恒定角速度绕太阳旋转（0.025-0.05 弧度/回合，每局随机）。可使用观测中的 `initial_planets` 与 `angular_velocity` 预测其位置。
- **静态行星**：距离中心更远的行星不会旋转。

地图包含 20-40 颗行星（5-10 组，每组 4 颗对称行星）。至少保证 3 组为静态行星，且至少有 1 组为轨道行星。

### 母星

会随机选择一组对称行星作为起始行星。2 人对局中，玩家出生在对角象限的两颗行星（Q1 和 Q4）。4 人对局中，每位玩家获得该组中的一颗行星。母星初始有 10 艘舰船。

## 舰队

每支舰队表示为 `[id, owner, x, y, angle, from_planet_id, ships]`。

- **angle**：移动方向（弧度）。
- **ships**：舰队中的舰船数量（航行过程中不会变化）。

### 舰队速度

舰队速度会随规模按对数曲线增长：

```
speed = 1.0 + (maxSpeed - 1.0) * (log(ships) / log(1000)) ^ 1.5
```

- 1 艘舰船时速度为 1.0 单位/回合。
- 舰队越大速度越快，并逐渐接近最大速度（默认 6.0）。
- 约 500 艘舰船时速度约为 5，约 1000 艘舰船达到上限。

### 舰队移动

舰队每回合按计算出的速度沿直线移动。若满足以下任一条件，舰队会被移除：

- 越界（离开 100x100 游戏区域）。
- 穿过太阳（路径线段进入太阳半径范围内）。
- 与任意行星碰撞（路径线段进入行星半径范围内），会触发战斗。

碰撞检测是连续的——检查的是从旧位置到新位置的整段路径，而不仅是终点。

### 舰队发射

每回合，你的智能体返回一个行动列表：`[from_planet_id, direction_angle, num_ships]`。

- 只能从你拥有的行星发射。
- 发射舰船数不能超过该行星当前驻军。
- 舰队会在给定方向上、行星半径之外紧邻处生成。
- 单回合内可从同一行星或不同行星发射多支舰队。

## 彗星

彗星是临时的太阳系外天体，会沿绕日高椭圆轨道穿越棋盘。它们按 4 颗一组生成（每象限 1 颗），生成回合为 50、150、250、350、450。

- **半径**：1.0（固定）。
- **产能**：被占领时每回合 1 艘舰船。
- **初始舰船数**：随机且偏小（从 1-99 中掷 4 次取最小）。同一组 4 颗彗星共享相同初始舰船数。
- **速度**：可通过 `cometSpeed` 配置（默认 4.0 单位/回合）。
- **识别方式**：在观测中的 `comet_planet_ids` 可查看哪些行星 ID 是彗星。彗星也会出现在 `planets` 数组中，并遵循所有普通行星规则（占领、产能、发射舰队、战斗）。

当彗星离开棋盘时，会连同其驻军一起被移除。每回合中，彗星移除发生在舰队发射之前，因此无法从即将离场的彗星发射舰队。

观测字段 `comets` 包含彗星组数据，其中包括 `paths`（每颗彗星的完整轨迹）和 `path_index`（当前轨迹位置索引），可用于预测未来位置。

## 回合顺序

每回合按以下顺序执行：

1. **彗星过期**：移除已离开棋盘的彗星。
2. **彗星生成**：在指定回合生成新的彗星组。
3. **舰队发射**：处理所有玩家动作，创建新舰队。
4. **产能结算**：所有已占领行星（包括彗星）生成舰船。
5. **舰队移动**：所有舰队沿航向移动。检查越界、太阳碰撞和行星碰撞。撞上行星的舰队会进入待战斗队列。
6. **行星旋转与彗星移动**：轨道行星旋转，彗星沿路径推进。任何被移动中的行星/彗星扫到的舰队都会被卷入与其的战斗。
7. **战斗结算**：结算所有排队中的行星战斗。

## 战斗

当一支或多支舰队与行星碰撞（主动飞入，或被移动中的行星扫到）时，按如下方式结算战斗：

1. 所有到达舰队按拥有者分组。同一拥有者的舰船数先求和。
2. 规模最大的进攻方与第二大的进攻方交战，差额舰船存活。
3. 若有进攻方幸存：
   - 若其拥有者与行星拥有者相同，则幸存舰船加入该行星驻军。
   - 若其拥有者不同，则幸存舰船与驻军交战。若进攻方超过驻军，行星易主，剩余舰船成为新驻军。
4. 若两方进攻舰队并列最大，则所有进攻舰船同归于尽（无幸存者）。

## 计分与终局

游戏在以下任一条件满足时结束：

- **达到回合上限**：500 回合。
- **淘汰终局**：仅剩一名（或零名）玩家仍拥有任意行星或舰队。

最终得分 = 所有己方行星驻军总数 + 所有己方舰队舰船总数。分数最高者获胜。

## 观测字段参考

| 字段 | 类型 | 说明 |
|-------|------|-------------|
| `planets` | `[[id, owner, x, y, radius, ships, production], ...]` | 所有行星（含彗星） |
| `fleets` | `[[id, owner, x, y, angle, from_planet_id, ships], ...]` | 所有活跃舰队 |
| `player` | `int` | 你的玩家 ID（0-3） |
| `angular_velocity` | `float` | 行星旋转速度（弧度/回合） |
| `initial_planets` | `[[id, owner, x, y, radius, ships, production], ...]` | 游戏开始时的行星位置 |
| `comets` | `[{planet_ids, paths, path_index}, ...]` | 当前活跃彗星组数据 |
| `comet_planet_ids` | `[int, ...]` | 属于彗星的行星 ID 列表 |
| `remainingOverageTime` | `float` | 剩余超时预算（秒） |

## 动作格式

返回一个行动列表：

```python
[[from_planet_id, direction_angle, num_ships], ...]
```

- `from_planet_id`：你拥有的行星 ID。
- `direction_angle`：弧度方向（0 = 向右，pi/2 = 向下）。
- `num_ships`：要派出的整数舰船数。

若本回合不行动，返回空列表 `[]`。

## 智能体开发便捷项

模块导出了命名元组，便于字段访问：

```python
from kaggle_environments.envs.orbit_wars.orbit_wars import Planet, Fleet, CENTER, ROTATION_RADIUS_LIMIT

def agent(obs):
    planets = [Planet(*p) for p in obs.get("planets", [])]
    fleets = [Fleet(*f) for f in obs.get("fleets", [])]
    player = obs.get("player", 0)

    for p in planets:
        print(p.id, p.owner, p.x, p.y, p.radius, p.ships, p.production)

    return []  # list of [from_planet_id, angle, num_ships]
```
