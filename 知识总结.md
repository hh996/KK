# Balanced Accuracy 知识总结

## 为什么需要 Balanced Accuracy

普通 Accuracy（准确率）：

`Accuracy = 正确预测的样本数 / 总样本数`

当数据集严重不平衡时（例如正样本 1%，负样本 99%），模型即使把所有样本都预测为负类，也能得到 99% 的 Accuracy，但这对少数类几乎没有识别能力。

Balanced Accuracy 通过给每个类别相同权重，避免“大类主导指标”，能更公平地反映模型在各类别上的真实表现。

## TP、TN、FP、FN 的意义

以“正类”为我们关心的目标（例如“需要灌溉”）：

- `TP`（True Positive，真正例）：实际为正，预测也为正
- `TN`（True Negative，真负例）：实际为负，预测也为负
- `FP`（False Positive，假正例）：实际为负，但预测为正（误报）
- `FN`（False Negative，假负例）：实际为正，但预测为负（漏报）
- `Recall_positive`（也叫 Sensitivity 或 TPR）= `TP / (TP + FN)`
- `Recall_negative`（也叫 Specificity 或 TNR）= `TN / (TN + FP)`

## 定义与公式（二分类）

### LaTeX 公式版（支持数学渲染时显示更好）

$$
\text{Balanced Accuracy}
= \frac{1}{2} \left( \text{Recall}_{\text{positive}} + \text{Recall}_{\text{negative}} \right)
= \frac{\text{TPR} + \text{TNR}}{2}
= \frac{1}{2} \left( \frac{TP}{TP + FN} + \frac{TN}{TN + FP} \right)
$$



## 指标范围与性质

- 取值范围：`0` 到 `1`（或 `0%` 到 `100%`）
- `1` 表示完美分类，`0` 表示最差
- 当数据集类别较平衡时，Balanced Accuracy 通常接近普通 Accuracy

## 多分类情形

对于多分类，Balanced Accuracy 定义为各类别 Recall 的宏平均（macro-average）：

`Balanced Accuracy = (各类别 Recall 的简单平均)`

即每个类别权重相同，不因样本数量不同而偏向大类。

# 📝 特征分布与变换笔记（总结版）

## 1. 什么是偏态（Skewness）？
*   **定义**：数据分布**不对称**。
*   **右偏（正偏）**：大多数数据小，少数极大值（长尾向右）。→ **收入、销量、点击量**（最常见）。
*   **左偏（负偏）**：大多数数据大，少数极小值（长尾向左）。→ **考试满分、年龄分布**。
*   **危害**：极端值干扰模型，导致线性模型、PCA、聚类等效果变差。

## 2. 什么时候用 Log 变换？
*   **核心作用**：压缩大值，拉伸小值，专治**右偏**。
*   **适用场景**：
    *   数据**明显右偏**（看图尾巴在右边）。
    *   数据范围**跨度极大**（比如 0 到 100万）。
    *   特征类型：**计数、金额、价格、功率、面积**。
    *   模型效果差（线性回归、聚类崩了）。
*   **禁忌**：**有负数**不能直接用 log。

## 3. 什么时候用 Box-Cox 变换？
*   **核心作用**：比 log 更强大的**自动正态化**工具（自动找最优参数 $\lambda$）。
*   **适用场景**：
    *   log 变换效果**不够好**时。
    *   你需要**严格接近正态分布**（统计建模、误差分析）。
    *   拿不准用 log 还是平方根时。
*   **禁忌**：**必须全为正数**。

## 4. 最终决策口诀（一手掌握）
1. **看图说话**：先画箱线图/直方图，看偏态方向。
2. **右偏首选**：先试 **Log**（简单高效）。
3. **Log 不行**：再上 **Box-Cox**（自动优化）。
4. **左偏/有负**：先处理数据（如取反），再用上述方法，或改用 Yeo-Johnson。
